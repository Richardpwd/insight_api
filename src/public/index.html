Endpoints

Uso da API

GET /health

Retorna o status da API. Nao requer autenticacao.

{ "status": "ok", "message": "Contract Insight API is running" }
POST /contracts/analyze

Analisa contrato em texto com IA (OpenAI) e fallback por regras. Requer header x-api-key.

{ "contractText": "...", "contractType": "prestacao_servico" }
POST /contracts/analyze/pdf

Upload de PDF via multipart. Campo file obrigatorio. Requer x-api-key.

Content-Type: multipart/form-data
file: contrato.pdf
contractType: prestacao_servico (opcional)
GET /contracts/history

Historico de analises em memoria. Requer x-api-key.

{ "history": [ { "id": 1, "riskScore": 68, ... } ] }
